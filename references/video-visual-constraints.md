# 视频视觉包装全局约束

本文件是 `video-prd-dual-agent` 的默认视觉约束。用于没有更高优先级品牌规范、参考图或用户指定风格时，为视频 PRD 提供统一的字幕、包装、动画和 HyperFrames 实现方向。

如果用户提供了明确品牌规范、参考视频、参考截图或平台模板，以用户素材为最高优先级；但仍应保留本文件中的信息清晰度、安全区、字幕可读性和包装不遮挡主体原则。

## 1. 总体风格

默认风格为：暗色视频口播 + 赛博 HUD 知识包装 + 双语字幕 + 技术报告感信息卡。

关键词：深色、技术感、AI 科技、Vibe Coding、HUD、网格卡片、风险印章、章节编号、数据仪表盘、冷光、红绿状态色。

视觉目标：真实口播视频仍是主体，包装层像一份技术分析报告叠加在画面上，而不是综艺贴纸、卡通花字或营销海报。

## 2. 画面基调

- 画面整体偏暗，人物保持中低亮度，背景偏冷色。
- 包装元素集中在左上、左中、右上和画面边缘，避免遮挡人物脸部、手势中心、产品主体和底部字幕。
- 背景色优先使用近黑、墨绿黑、蓝黑、暗紫黑。
- 主文字使用高对比白色。
- 危险、淘汰、否定、警告信息使用霓虹红或暗红。
- 免疫、护城河、通过、稳定信息使用荧光绿或深绿色。
- 根因、解释、结构分析使用科技蓝。
- 金额、排行、关键数据使用金黄或琥珀色。

## 3. 字幕约束

底部默认使用双语字幕：

- 中文为主行，白色粗体，居中，字号大。
- 英文为副行，白色小号，位于中文下方，字重较轻。
- 字幕可使用轻微黑色阴影或 30%-45% 透明黑色背景条。
- 字幕宽度控制在画面中下方 60%-70%，避免贴边。
- 每屏不超过 2 行中文；竖屏优先不超过每行 14 个中文字符。
- 不频繁做关键词彩色高亮，重点信息优先交给 HUD 卡片、标题和标签承担。

## 4. 包装组件

### 左上章节 HUD

- 格式示例：`| IMMUNE · 01`、`| DISSOLVED · 03`、`| ROOT CAUSE · 根因`、`| FORECAST · 2-3 YEARS`。
- 英文大写，字距拉开，字号小，使用章节状态色。
- 下方配一行中文短解释。

### 中左主标题

- 中文超粗白字，通常 2-6 个字。
- 下方配小号英文副标题，如 `PERMISSION GATE`、`LIABILITY BEARER`。
- 可附带小图标、小状态标签、短横线或概念解释标签。

### 技术图标卡

- 方形网格卡片，使用绿色、蓝色或红色 glow。
- 内部使用线性图标，如钥匙、天平、数据库、闪电、脑、链路。
- 边缘使用细描边、暗色玻璃质感和轻微发光。

### 风险印章

- 红色线框章，倾斜 5-12 度。
- 示例文字：`判死刑 / DEATH SENTENCE`、`NO MOAT?`。
- 用于否定、淘汰、警告或结论反转。

### 案例和数据仪表盘

- 左侧竖向资料卡可包含公司名、案例编号、金额、排行、进度条和图标按钮。
- 右侧可放网页、产品或数据截图，使用大圆角白色面板。
- 信息密度可以高，但排版必须克制、分组清晰。

### 小型解释标签

- 半透明黑底，小圆角，左侧小图标。
- 用于补充概念、条件、结论和风险提示。
- 位置靠近主标题或手势区域，但不遮脸。

## 5. 动画约束

整体节奏：

- 动画跟随口播观点点出现，不持续乱动。
- 每个新概念出现时使用 0.2-0.45 秒短促入场。
- 入场完成后只保留轻微呼吸、扫描线或 glow。
- 避免持续大幅漂浮、无意义旋转和抢主体的大面积动效。

组件动画：

- 章节 HUD：从左侧轻微滑入，透明度 0 到 1，字符间距打开。
- 主标题：快速上滑或左滑入，scale 0.96 到 1，带轻微 overshoot。
- 英文副标题：延迟 0.08-0.15 秒淡入，像系统标签加载。
- 网格图标卡：网格逐格点亮，图标 pop in，外圈 glow 闪一下。
- 风险印章：旋转盖章式弹入，scale 1.15 到 1，伴随短暂红光闪烁。
- 数据进度条：从左到右填充，数字轻微滚动。
- 截图或网页面板：从右侧或下方滑入，带轻微模糊到清晰。
- 章节切换：使用横向扫描线、暗场闪白或色彩 tint，不使用花哨转场。

## 6. 生成提示词模板

为横屏或竖屏真人口播视频设计暗色赛博科技 HUD 字幕包装。保留真人口播为主体，画面整体压暗，使用蓝黑、墨绿黑、暗紫黑背景，高对比白字。底部固定双语字幕：中文粗体大字在上，英文小号白字在下，带轻微阴影或半透明黑底。

左上加入章节 HUD 标签，中左加入超粗中文概念标题和英文副标题，配合网格图标卡、红色风险印章、半透明解释标签、数据仪表盘、网页截图面板。根据语义搭配状态色：危险/淘汰用红色，护城河/免疫用绿色，根因解释用蓝色，数据重点用金黄色。

动画短促、克制、跟随口播观点点：标题滑入缩放，网格逐格点亮，印章盖章弹入，数据条填充，截图面板模糊滑入。每个元素入场 0.2-0.45 秒，停留时只有轻微 glow、扫描线或呼吸感。整体像专业 AI 技术分析频道，不要综艺花字、卡通贴纸、过度可爱、大面积遮挡主体或伪造事实画面。

## 7. HyperFrames 实现约束

- 视频为主轨，包装层为透明覆盖层。
- 固定字幕组件：中文 42-56px 粗体白字，英文 18-24px 白字，半透明黑色背景条。
- 可复用 overlay 组件建议：`chapterHud`、`conceptTitle`、`gridIconCard`、`riskStamp`、`dataPanel`、`screenshotPanel`、`microLabel`。
- 使用 GSAP 同步口播时间点。
- `chapterHud` 使用 `opacity + x + letterSpacing` 入场。
- `conceptTitle` 使用 `y/x + opacity + scale` 入场。
- `gridIconCard` 使用 stagger 点亮格子和图标 pop。
- `riskStamp` 使用 `rotation + scale + opacity` 的盖章动效。
- `dataPanel` 使用进度条 width 动画和数字滚动。
- `screenshotPanel` 使用 `x/y + opacity + blur` 入场。
- 所有 hold motion 必须有限循环，不使用无限 `repeat: -1`。
- 包装位置必须避开人物脸、手势中心、产品主体和底部字幕安全区。
